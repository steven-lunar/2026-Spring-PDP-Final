2. Architectural Analysis: Resolving the Memory Bottleneck Through ILP
Understanding the False Paradigm of Vector "Fusion": The previous iteration attempted to save DRAM bandwidth by fusing the vector operations. However, the profiling data revealed a critical insight: Total Read Volume remained essentially identical at 570 GB. Fusing vectors q, d, z, and r only saves fractions of a Megabyte per loop. Mathematically, the 41.5 Million non-zeros in the sparse matrix A and colidx arrays dictate over 99% of the memory footprint. Fusing the scalar output of q into d did nothing to alleviate the primary bottleneck: the latency starvation caused by indirect sparse lookups.

The Root Cause of Low IPC (1.214): In Class C (nonzer = 15), each row contains roughly 15 non-zero elements. When a compiler encounters a loop iterating 15 times with an indirect read (p(colidx(k))), it panics. It attempts to deploy 512-bit AVX-512 SIMD (which operates in 8-lane chunks for FP64), resulting in one clean 8-lane fetch, one heavily masked 7-lane fetch, and substantial loop-control branching overhead. The processor execution units become choked on calculating vector masks and resolving loop iterations rather than performing useful FMA operations.

The Unrolling Solution: To fix this, I completely restructured the two SpMV q = A.p and r = A.z execution blocks by manually unrolling the inner matrix dimension by a factor of 8. This provides exactly what the Ice Lake-SP cores require:

Instruction-Level Parallelism (ILP): Instead of looping, the processor is fed 8 distinct, parallel scalar/vector FMA instructions sequentially. This immediately bypasses SIMD masking overhead and fills the execution ports, ensuring high IPC.

Hardware Prefetching Synergy: By statically defining 8 parallel lookups a(i+0) * p(colidx(i+0)) through a(i+7) * p(colidx(i+7)), the processor's native hardware prefetcher can "see" the next 8 memory addresses instantly and pipeline the memory requests. This hides the DRAM latency of the indirect gathers inside the L1/L2 caches, resolving the starvation at the memory controller.
